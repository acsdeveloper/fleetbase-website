Organisation Settings & Permissions

Organisation settings and permissions give you complete control and visibility over who can access what data and what they can do with it. The role management dashboard lists all roles (admin, manager, dispatcher, driver, technician, finance), and click any role to see exactly which permissions are assigned (read, edit, delete, approve) for each feature. Below that is a user management grid showing all users with their assigned role(s). Bulk actions let you change roles for multiple users at once (e.g., promote 5 dispatchers to manager), and you can search, sort, and filter by role, department, or status. The permission editor gives you granular control over what each role can do. A dispatcher can create and assign trips but cannot delete them. Finance can approve expenses but cannot modify trip data. Custom roles are supported for roles outside the standard templates. An audit log viewer shows the complete change history: who changed what, when, and from which device. Filter by user, feature, date, or action type (create, edit, delete, approve), and export the log for compliance audits. For enterprise organisations, SSO and directory integration connects to Okta, Azure AD, or Google Workspace so users sign in with their company credentials-no separate FleetYes password to remember or reset. Roles are automatically assigned based on directory attributes (department, job title). API key management lets you create keys for third-party integrations, scoped to specific permissions (e.g., read-only trip data), and rotate them regularly for security. Session and login monitoring shows who's logged in, which device they're using, and from which location, and you can force logout if needed or set automatic timeout after inactivity. For scaling, bulk user import lets you import users from a CSV or directory and automatically assign roles based on department or job title, provisioning dozens of users in seconds.

Why Choose Organisation Settings & Permissions
