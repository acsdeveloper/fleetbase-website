Image Processing

When the camera opens, a native camera interface appears with preview, capture button, and flash control. After capturing, a preview screen shows the image with metadata overlays (GPS coordinates, timestamp, accuracy). The driver can retake the photo or confirm. Confirmed photos appear in a gallery with thumbnails. Processing happens in the background—the UI remains responsive while compression and EXIF extraction occur.

Why Choose Image Processing

Image processing seems simple—take a photo, upload it. But the details matter enormously. Unoptimised images are huge (5-10MB), slow to upload, and blow through data plans. Automated processing reduces them to 200-400KB without visible quality loss.

EXIF metadata is the difference between a photo that proves something and a photo that's just evidence. A delivery photo with GPS coordinates and timestamp that matches the order assignment time is nearly irrefutable proof. A photo without that metadata is just a picture.

Handling both camera and gallery import is important. Sometimes drivers need to use existing photos. Supporting both paths makes the feature flexible without requiring multiple workflows.
